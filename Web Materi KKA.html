Intro

Pendahuluan

Mata pelajaran Koding dan Kecerdasan Artifisial (KKA) dirancang untuk membekali peserta didik dengan kemampuan berpikir komputasional, keterampilan pemrograman dasar, serta pemahaman rasional dan kritis tentang kecerdasan artifisial (AI). Buku ini disusun sebagai buku ajar lengkap yang dapat digunakan secara mandiri maupun dalam pembelajaran di kelas.

Tujuan utama buku ini bukan sekadar membuat siswa bisa menulis kode, tetapi memahami logika di balik teknologi, mampu menganalisis masalah, dan menyadari dampak AI terhadap kehidupan, etika, serta masa depan manusia.

BAB 1 – PENGANTAR KODING DAN KECERDASAN ARTIFISIAL

1.1 Pengertian Koding

Koding adalah proses menuliskan instruksi secara sistematis menggunakan bahasa tertentu agar komputer dapat menjalankan perintah sesuai keinginan manusia. Komputer tidak berpikir seperti manusia; ia hanya mengeksekusi instruksi secara logis, berurutan, dan tanpa asumsi. Oleh karena itu, koding menuntut ketelitian, konsistensi, dan kejelasan logika.

1.2 Pengertian Kecerdasan Artifisial (AI)

Kecerdasan Artifisial adalah cabang ilmu komputer yang mempelajari cara membuat sistem atau mesin yang mampu meniru kemampuan kognitif manusia. AI bekerja berdasarkan data, algoritma, dan model matematika.

1.3 Hubungan Koding dan AI

AI tidak mungkin ada tanpa koding. Koding adalah fondasi dari semua sistem AI.

1.4 Tujuan Pembelajaran KKA

BAB 11 – KODING SECARA MENDALAM: CARA BERPIKIR SEORANG PROGRAMMER

11.1 Koding Bukan Mengetik Kode

Kesalahan paling umum dalam pembelajaran koding adalah mengira bahwa koding berarti mengetik sintaks. Itu keliru. Mengetik hanyalah tahap akhir. Inti koding adalah menyusun logika yang tidak ambigu. Programmer yang baik menghabiskan lebih banyak waktu berpikir dibandingkan menulis kode.

Setiap baris kode adalah keputusan logis. Jika keputusan itu salah, program tetap berjalan tetapi menghasilkan keluaran yang keliru. Inilah mengapa debugging adalah bagian penting dari koding.

11.2 Problem Solving dalam Koding

Setiap program selalu berangkat dari masalah nyata. Tahapan rasionalnya:

  1. Memahami masalah secara utuh
  2. Menentukan input dan output
  3. Menyusun algoritma
  4. Baru menuliskan kode

Melompati tahapan ini adalah kebiasaan buruk yang membuat siswa tergantung pada contoh, bukan pemahaman.

11.3 Debugging dan Error

Error terbagi menjadi:

  • Syntax Error: kesalahan penulisan
  • Runtime Error: kesalahan saat program berjalan
  • Logical Error: hasil salah tetapi tidak ada pesan error

Kesalahan logika adalah yang paling berbahaya karena sering tidak disadari.

12.1 Mengapa Struktur Data Penting

Struktur data adalah cara mengatur data agar efisien dan logis. Program tanpa struktur data yang tepat akan lambat dan sulit dikembangkan.

12.2 Jenis Struktur Data Dasar

  • Array/List: menyimpan banyak data sejenis

Pemilihan struktur data yang salah adalah kesalahan berpikir, bukan kesalahan teknis.

12.3 Hubungan Struktur Data dan AI

AI modern sangat bergantung pada pengelolaan data dalam jumlah besar. Tanpa struktur data, AI tidak mungkin bekerja.

BAB 16 – PRAKTIK KODING DASAR DENGAN PYTHON

16.1 Alasan Menggunakan Python

Python dipilih karena sintaksnya mendekati bahasa manusia, sehingga siswa dapat fokus pada logika, bukan menghafal aturan bahasa. Python juga digunakan secara luas dalam pengembangan web, analisis data, dan kecerdasan artifisial. Ini berarti keterampilan yang dipelajari relevan dengan dunia nyata, bukan simulasi akademik kosong.

Python bersifat interpreted, artinya kode dijalankan baris demi baris. Konsekuensinya jelas: setiap kesalahan logika langsung berdampak pada hasil. Ini sangat baik untuk melatih ketelitian berpikir siswa.

16.2 Struktur Program Python

Setiap program Python memiliki alur yang konsisten: menerima input, memproses data, dan menghasilkan output. Jika salah satu bagian ini tidak jelas, program akan gagal secara fungsional.

Contoh konsep (bukan untuk dihafal, tapi dipahami):

nama = input("Masukkan nama: ")
print("Halo", nama)

Kode di atas menunjukkan bahwa komputer tidak memahami konteks sosial. Semua harus dinyatakan eksplisit.

16.3 Variabel dan Tipe Data dalam Python

Variabel adalah identitas bagi data. Tanpa variabel, data tidak bisa dikelola. Python bersifat dynamic typing, tetapi itu bukan alasan untuk berpikir sembarangan.

Contoh:

umur = 17      # int
berat = 55.5   # float
nama = "Ani"  # str
lulus = True   # bool

Kesalahan umum siswa adalah mencampur tipe data tanpa memahami konsekuensinya, misalnya menjumlahkan teks dengan angka.

BAB 17 – LOGIKA, KONDISI, DAN PERULANGAN (PRAKTIK)

17.1 Percabangan if–elif–else

Percabangan digunakan ketika program harus mengambil keputusan. Komputer tidak menebak; ia hanya membandingkan kondisi.

Contoh:

nilai = 80
if nilai >= 75:
    print("Lulus")
else:
    print("Tidak lulus")

Jika kondisi salah dirumuskan, hasilnya pasti salah walaupun kodenya berjalan.

17.2 Perulangan for dan while

Perulangan digunakan saat suatu proses dilakukan berkali-kali. Tanpa perulangan, kode akan berulang dan tidak efisien.

Contoh for:

for i in range(5):
    print(i)

Contoh while:

i = 0
while i < 5:
    print(i)
    i += 1

Kesalahan paling fatal: Lupa kondisi berhenti.

17.3 Studi Kasus

Studi kasus digunakan untuk melatih siswa menerjemahkan masalah nyata ke dalam logika program, bukan menyalin kode.

BAB 21 – DASAR-DASAR PEMBUATAN WEBSITE

21.1 Apa Itu Website (Tanpa Ilusi)

Website adalah kumpulan halaman yang diakses melalui internet menggunakan browser. Website bukan sekadar tampilan, tetapi sistem yang menyajikan informasi secara terstruktur.

Kesalahan umum siswa adalah mengira website = desain. Ini keliru. Desain tanpa struktur adalah hiasan kosong.

21.2 Cara Kerja Website

Website bekerja melalui mekanisme client–server:

  • Client (browser) meminta data
  • Server memproses permintaan
  • Server mengirimkan respons

Jika siswa tidak memahami alur ini, maka pembuatan website hanya akan menjadi aktivitas meniru template.

21.3 Komponen Utama Website

  1. HTML – struktur dan makna konten
  2. CSS – tata letak dan tampilan
  3. JavaScript – logika dan interaksi

Ketiganya memiliki fungsi berbeda. Mencampur peran mereka adalah kesalahan konseptual.

BAB 22 – HTML: STRUKTUR LOGIS WEBSITE

22.1 HTML sebagai Bahasa Struktur

HTML bukan bahasa pemrograman, melainkan bahasa markup. Fungsinya memberi makna pada konten.

Contoh struktur dasar:

<!DOCTYPE html>
<html>
  <head>
    <title>Website Pertama</title>
  </head>
  <body>
    <h1>Judul</h1>
    <p>Paragraf</p>
  </body>
</html>

Setiap tag memiliki fungsi. Menggunakan tag tanpa memahami maknanya adalah kebiasaan buruk.

22.2 Kesalahan Umum HTML

  • Menggunakan div untuk semua hal
  • Tidak memahami hierarki tag
  • Mengabaikan semantik

Kesalahan ini menunjukkan lemahnya pemahaman struktur.
